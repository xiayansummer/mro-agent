**日期**：2026-03-27
**状态**：已确认，待实施

## 一、产品定位

**产品名**：FA 秘书

**核心定位**：每位 FA 专属的 AI 工作助理，语音优先，随时随地，越用越懂你。

**目标用户**：FA（财务顾问）内部使用，不对外展示。每位 FA 绑定独立的微信账号（openid），拥有独立的记忆和偏好。

**核心体验循环**：

```
FA 开口说话
  → 小程序录音 → STT 转文字
  → Agent 理解意图 → 调用工具（查投资人 / 写材料 / 查项目 / 记备注）
  → 生成回复 → TTS 播报 + 文字 + 卡片展示
  → FA 追问或确认
  → 行为沉淀为个人偏好记忆
```

## 二、功能范围

### Phase 1（当前）

#### A. 投资人匹配（高优先级）

- FA 语音描述融资项目需求（赛道、阶段、金额、地域等）
- Agent 解析意图，调用 `search_investors()` 工具查询 MySQL 投资人库
- 返回：结构化投资人/机构卡片列表（按匹配度排序）+ AI 语音解释匹配理由
- 支持追问细化："有没有更偏好消费赛道的"、"上次那种风格的"
- 每次推荐结果记录 FA 的选择行为，用于后续偏好加权

#### B. 材料生成（高优先级）

- 支持起草：项目介绍邮件、路演 BP 摘要、投资人沟通话术
- FA 语音说明需求，Agent 生成草稿，支持语音追问修改
- 生成结果可保存为 Memo，关联项目和投资人

#### C. 日历与待办（Phase 1 内置）

- Agent 识别对话中的 to-do 意图，自动创建日历事项
- 小程序内置日历组件（不跳出），数据存 MySQL
- 微信订阅消息在事项到期前推送提醒
- 日历事项可关联具体项目和投资人

#### D. 备忘录（Memo）

- FA 随时记录碎片信息："记一下，张总对新能源态度保守"
- 存入 MySQL `memos` 表，自动打上项目/投资人标签
- Agent 可检索历史备忘："我之前对张总有什么备注？"

### Phase 2（后续迭代）

- **C. 全流程管理**：项目从录入 → 投资人匹配 → 触达跟进 → 状态追踪
- **历史案例**：接入已投项目数据，优化匹配算法
- **团队协作**：多 FA 共享项目，权限管理

### 技术选型

| 模块 | 技术 | 理由 |
|------|------|------|
| 小程序框架 | 原生微信小程序（WXML/WXSS/JS） | 录音/订阅消息 API 最稳定 |
| STT | 阿里云实时语音转写 / 讯飞 | 中文金融场景准确率高 |
| TTS | 阿里云 TTS / 讯飞 | 语音自然，支持多音色 |
| Agent | Claude API（tool_use） | 工具调用能力强，推理质量高 |
| 流式输出 | WebSocket | 小程序对 SSE 支持有限 |
| 后端框架 | FastAPI（复用 mro-agent 模式） | 团队已熟悉，部署一致 |
| 推送提醒 | 微信订阅消息 | 官方推送，不跳出小程序 |
| 数据库 | MySQL（复用现有） | 投资人/项目数据已在其中 |

## 六、个人化记忆机制

**越用越顺手的实现方式**：

1. **显式反馈**：FA 选择某个投资人推荐 → 记录正向信号；跳过某人 → 负向信号
2. **隐式偏好**：统计 FA 常关注的机构类型、赛道、投资阶段
3. **风格记忆**：FA 修改 Agent 生成的材料 → 学习表达偏好（正式/简洁/详细）
4. **关系记忆**：FA 对某投资人的备忘自动影响下次匹配权重

每次 `search_investors()` 调用时，自动读取该 FA 的 `fa_memory` 并注入 Agent 的系统提示。
